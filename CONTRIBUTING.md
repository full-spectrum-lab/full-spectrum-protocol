# Contributing

Thank you for considering a contribution to the Full Spectrum Ethics Protocol Stack.

This project is an early protocol draft. Contributions are welcome, but protocol-level changes should be careful, traceable, and reviewable.

## What contributions are welcome?

### Documentation

- Fix encoding, typos, broken links, and unclear explanations.
- Improve Chinese and English readability.
- Add glossary entries.
- Add diagrams or structured reading paths.

### Protocol critique

- Identify unclear concepts.
- Point out abuse cases.
- Point out contradictions.
- Propose safer wording.
- Compare with existing AI governance, safety, and ethics frameworks.

### Engineering

- Add JSON schemas.
- Add minimal examples.
- Add testable use cases.
- Add audit-trace examples.
- Add FSHI sample input/output files.

### Translation

Chinese and English translations are especially valuable.

## Contribution flow

1. Fork the repository.
2. Create a branch.
3. Make changes.
4. Open a pull request.
5. Explain the motivation and risk of the change.

Example:

```bash
git checkout -b docs/fix-readme
git add .
git commit -m "docs: fix README entry and reading path"
git push origin docs/fix-readme
```

## Protocol-level changes

Protocol-level changes should include:

1. Problem statement.
2. Proposed change.
3. Affected modules.
4. Abuse and failure cases.
5. Compatibility or rollback plan.
6. Examples or counterexamples.

## Conduct

Please follow [CODE_OF_CONDUCT.md](./CODE_OF_CONDUCT.md).

Disagreement is welcome. Personal attacks are not.
