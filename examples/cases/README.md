# Synthetic CASE Examples

This directory contains synthetic examples for discussing and testing Full Spectrum Protocol concepts.

---

## 1. Important boundary

All CASE documents in this repository are synthetic unless explicitly stated otherwise.

They do not represent:

- real enterprise data;
- real customer conversations;
- real customer deployments;
- authorized statements from any company;
- regulatory conclusions;
- production validation.

They are structured examples for protocol design, schema testing, and governance reasoning.

---

## 2. What CASE examples are for

CASE examples help test whether the protocol can express:

- conflicting AI outputs;
- unclear responsibility;
- risk accumulation;
- boundary crossing;
- human review needs;
- cross-system audit records;
- knowledge-source conflict;
- non-invasive enterprise inspection.

---

## 3. Recommended CASE structure

Each CASE should include:

1. scenario summary;
2. participating subjects or nodes;
3. source of conflict;
4. Governance Event;
5. RiskAlert;
6. RiskVector if applicable;
7. AuditTrace;
8. review recommendation;
9. protocol boundary note;
10. synthetic-data declaration.

## 5. Contribution rule

Do not submit real customer data.

Use synthetic, desensitized, or fictionalized examples. If an example is inspired by a real scenario, remove identifying details and state that it is a synthetic reconstruction.
