# v1.0-beta Release Criteria

This document defines the minimum conditions before Full Spectrum Protocol should be presented as a public v1.0-beta protocol draft.

Current status: pre-beta.

---

## 1. What v1.0-beta should mean

v1.0-beta does not mean:

- finished standard;
- regulatory approval;
- production guarantee;
- commercial product maturity;
- complete solution to AI governance.

v1.0-beta means:

- the public entry is understandable;
- the core objects have stable draft schemas;
- synthetic examples can be validated;
- a minimal local-first engine can run;
- contribution and review paths are clear;
- external readers can distinguish protocol, engine, examples, and private products.

---

## 2. Required public documents

Before v1.0-beta, the repository should contain:

- [x] README with clear positioning and boundary;
- [x] FAQ;
- [x] Protocol Boundary Statement;
- [x] Minimal Integration Guide;
- [x] Governance Event minimal specification;
- [x] RiskAlert specification or RFC;
- [x] AuditTrace specification or RFC;
- [x] Cross-Enterprise Audit Record profile or RFC;
- [x] External Node Guide;
- [x] Standards and ecosystem mapping;
- [x] Conformance and testing guide;
- [x] Roadmap.

---

## 3. Required schemas

Before v1.0-beta, the repository should contain machine-readable schemas for:

- [x] GovernanceEvent;
- [ ] IdentityClaim;
- [ ] CapabilityDeclaration;
- [x] RiskAlert;
- [x] AuditTrace;
- [x] CrossEnterpriseAuditRecord;
- [x] FSHI dialogue inspection request;
- [x] FSHI dialogue inspection response.

Each schema should have at least one synthetic sample.

---

## 4. Required examples

Before v1.0-beta, examples should include:

- [x] one minimal GovernanceEvent sample;
- [x] one RiskAlert sample;
- [x] one AuditTrace sample;
- [x] one CrossEnterpriseAuditRecord sample;
- [x] one FSHI dialogue inspection sample chain;
- [ ] at least three synthetic CASE documents;
- [x] clear statement that examples are synthetic and do not represent real enterprise data.

---

## 5. Required validation

Before v1.0-beta:

- [x] mojibake check passes;
- [x] FSHI contract chain validation passes;
- [x] GovernanceEvent sample validation passes;
- [ ] JSON samples are valid;
- [x] GitHub Actions runs on push and pull request;
- [ ] documentation links are reviewed.

---

## 6. Required engine relationship

Before v1.0-beta, the public protocol repository should clearly link to the minimal engine and explain:

- the engine is a reference implementation;
- the engine is local-first;
- the engine does not require joining a protocol network;
- the engine does not replace legal or organizational review;
- the engine should be tested with synthetic or desensitized data first.

---

## 7. Required boundary statements

Before v1.0-beta, public materials must avoid claiming:

- real enterprise production adoption unless authorized;
- regulatory recognition;
- legal effect;
- final responsibility adjudication;
- automatic safety guarantee;
- universal applicability to all industries.

---

## 8. Recommended v1.0-beta tag wording

Recommended:

```text
v1.0-beta: open governance semantics protocol draft
```

Not recommended:

```text
official AI governance standard
complete AI safety protocol
production-ready enterprise compliance system
```

---

## 9. Exit condition

The project is ready for v1.0-beta when a new reader can:

1. understand what Full Spectrum Protocol is in 10 minutes;
2. run or inspect one minimal example in 30 minutes;
3. validate at least one protocol object;
4. understand what the project does not claim;
5. know how to submit an issue, counterexample, schema improvement, or synthetic case.
